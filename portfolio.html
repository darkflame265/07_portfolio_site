  • 실시간 이벤트 처리 지연 감소 및 입력 반응 속도 개선
  • 구조 단순화를 통한 안정적인 실시간 동기화 달성

Problems & Solutions
PROBLEM 01 CNN 기반 학습 성능 한계, MLP 기반 학습 전환
CNN 구조 (Before)
이미지 기반 학습
(연속된 이미지로 정보 습득)
MLP 구조 (After)

문제
  • CNN 기반 이미지 입력 구조로 탄막 회피 AI 학습 진행
  • 장시간 학습에도 회피 성능이 개선되지 않는 문제 발생
  • 이미지 기반 고차원 입력으로 학습 효율이 낮은 구조적 한계 확인

  • 이미지 입력 대신 상태 기반 피처를 활용한 MLP 구조로 전환
  • 저차원 상태 입력 기반 강화학습 환경으로 재구성
결과
  • MLP 기반 학습에서 안정적인 회피 행동 학습 확인
  • CNN 대비 학습 효율 및 성능 개선 달성
  • 모델 선택과 입력 구조 설계의 중요성 경험 확보
PROBLEM 02 CPU 단일 환경 학습 병목, GPU 병렬 환경 학습 전환
CPU 기반 단일 환경 학습 (Before)
학습 속도는 빠르지만, 한 번에 하나의
에이전트만 학습되어 속도 병목 발생
GPU 기반 병렬 환경 학습 (After)
학습 속도는 느리지만, 32개의 환경을 동시에
학습하여 속도 병목 해결
문제
  • CPU 기반 단일 환경 학습 구조로 전체 학습 속도 저하
  • 한 번에 하나의 에이전트만 학습되어 속도 병목 발생
  • 장시간 학습에도 충분한 학습 스텝 확보가 어려운 문제
해결
  • CUDA 기반 GPU 연산을 활용한 학습 구조로 전환
  • 벡터 환경(vec env) 도입으로 32개 환경 병렬 강화학습 구현
  • 32개 환경 동시 실행 학습 파이프라인 재구성
결과
  • 학습 속도 약 30배 이상 향상
  • 수십만 스텝 이상의 대규모 학습 수행 가능
  • 병렬 강화학습 및 학습 인프라 최적화 경험 확보
